# SA-MP 0.3.7-R5 map-icon compatibility

## Evidence

- Original DLL: `samp.dll` 0.3.7-R5
- SHA256: `b72b5dbe725f81864ca3f78bc7063bda56cc05fc7188af822fa7a754432553a2`
- Evidence classification: `STATIC_037`

The following behavior was recovered by static analysis of the original R5
DLL. Runtime comparison remains useful for visual confirmation, but the wire
layout, slot count, and command selection below are no longer inferred.

## RPC wire layouts

### RPC 56: SetPlayerMapIcon

Handler: `samp.dll+0x1A790`

The payload is exactly 19 bytes in this order:

| Offset | Type | Field |
| ---: | --- | --- |
| 0 | `uint8` | slot index |
| 1 | `float[3]` | X, Y, Z |
| 13 | `uint8` | icon |
| 14 | `uint32` | color |
| 18 | `uint8` | style |

The handler forwards all seven values to the map-icon manager. The original
manager accepts indices 0 through 99.

### RPC 144: RemovePlayerMapIcon

Handler: `samp.dll+0x1A8B0`

The payload is one `uint8` slot index. RPC 144 has no clock-toggle behavior;
clock toggling belongs to RPC 30.

## Original map-icon manager

- Set/replace slot: `samp.dll+0x0A300`
- Disable one slot: `samp.dll+0x08FB0`
- Clear all 100 slots: `samp.dll+0x0A2C0`
- Create GTA marker: `samp.dll+0x0A0D90`

Setting an occupied slot first disables its previous GTA marker. The marker
creation opcode is selected by style:

| Style | Meaning | GTA opcode |
| ---: | --- | ---: |
| 0 | local | `0x04CE` |
| 1 | global | `0x02A8` |
| 2 | local checkpoint | `0x0570` |
| 3 | global checkpoint | `0x02A7` |

Unknown styles take the style-0 branch in the original function.

For icon 0 only, the original then applies the wire color unchanged with
opcode `0x0165`. It applies scale 2 when the numeric color value is below 1004
and scale 3 otherwise, using opcode `0x0168`. Named GTA sprite icons skip both
of these overrides. Removal uses opcode `0x0164`.

## Replacement implementation

The replacement now preserves all 100 logical slots, all four styles, exact
replacement/removal lifecycle, icon-0 color/scale behavior, and pool cleanup
on reconnect, game-mode restart, or network disconnect. Its 128-entry
transition ring can carry a complete 100-slot refresh between runtime
snapshots. Decoder traces include index, icon, color, style, and position for
direct original/replacement run comparison.
